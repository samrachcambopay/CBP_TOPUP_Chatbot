# Cambopay TOPUP Chatbot — Python FastAPI Backend

FastAPI + MongoDB Atlas backend. All API docs (endpoints, parameters, response codes, topup amounts, signatures, SOAP samples) are stored in MongoDB and fetched dynamically to build the AI system prompt.

---

## 📁 Project Structure

```
cambopay-fastapi/
├── main.py              ← FastAPI app + all routes + system prompt builder
├── database.py          ← MongoDB Atlas async connection (Motor)
├── seed.py              ← Populate MongoDB with all Cambopay API data
├── models/
│   └── schemas.py       ← Pydantic models for all collections
├── requirements.txt     ← Python dependencies
├── .env.example         ← Copy to .env and fill in your values
└── README.md
```

## ⚡ Quick Start

### 1. Create virtual environment
```bash
python -m venv venv
source venv/bin/activate        # Mac/Linux
venv\Scripts\activate           # Windows
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Setup environment variables
```bash
cp .env.example .env
# Edit .env — add your MongoDB URI and Anthropic API key
```

### 4. Setup MongoDB Atlas (Free)
1. Go to https://cloud.mongodb.com → Sign up free
2. Create a **Cluster** (free M0 tier)
3. Create a **Database User** (username + password)
4. **Network Access** → Add IP: `0.0.0.0/0`
5. **Connect** → **Drivers** → choose **Python** → copy the URI
6. Paste into `.env` as `MONGODB_URI` (replace `<password>`)

### 5. Seed MongoDB with all API data
```bash
python seed.py
```
This populates 6 collections:
- `endpoints` — UAT + Production URLs (12 records)
- `parameters` — Request/Response params for all APIs
- `response_codes` — Error codes with refund rules
- `topup_amounts` — Operator limits and denominations
- `signatures` — RSA sign format per API
- `soap_samples` — Full SOAP XML request examples

### 6. Run the server
```bash
# Development
uvicorn main:app --reload --port 3000

# Production
uvicorn main:app --host 0.0.0.0 --port 3000
```

API docs (auto-generated by FastAPI):
- Swagger UI: http://localhost:3000/docs
- ReDoc:       http://localhost:3000/redoc

## 🌐 API Routes

| Method | Route | Description |
|--------|-------|-------------|
| GET  | `/` | Health check |
| POST | `/api/chat` | AI chat — builds prompt from MongoDB |
| GET  | `/api/endpoints` | All API URLs (filter: environment, service) |
| GET  | `/api/parameters/{api}` | Parameters for an API (filter: direction) |
| GET  | `/api/response-codes` | Response codes (filter: api, code) |
| GET  | `/api/topup-amounts` | Topup limits (filter: type, operator) |
| GET  | `/api/signatures` | RSA signature guide (filter: api) |
| GET  | `/api/soap-samples/{api}` | SOAP XML samples |

## 📡 Chat API Usage

**POST** `/api/chat`

Request:
```json
{
  "environment": "UAT",
  "messages": [
    { "role": "user", "content": "How do I do a Pinless topup?" }
  ]
}
```

Response:
```json
{
  "reply": "To perform a Pinless TOPUP, send a SOAP request to..."
}
```

## 🔧 Update Frontend HTML

In `cambopay-chatbot.html`, the `BACKEND_URL` is already set. After deploying, update it:

```javascript
// Change this line:
const BACKEND_URL = 'http://localhost:3000';

// To your deployed backend URL:
const BACKEND_URL = 'https://cambopay-api.onrender.com';
```

The `sendMsg()` function already calls `POST /api/chat` on your backend — no other changes needed.

## 🚀 Deploy to Render.com (Free — Recommended for Python)

1. Push to GitHub:
```bash
git init && git add . && git commit -m "init"
git remote add origin https://github.com/yourname/cambopay-fastapi.git
git push -u origin main
```

2. Go to https://render.com → **New Web Service** → connect your repo

3. Settings:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

4. **Environment Variables** → Add:
   ```
   MONGODB_URI=mongodb+srv://...
   ANTHROPIC_API_KEY=sk-ant-...
   FRONTEND_URL=https://your-frontend.netlify.app
   ```

5. Deploy → get URL like `https://cambopay-api.onrender.com`

## 🔐 Security Notes
- Never commit `.env` to Git
- API key lives on server only — never exposed to browser
- CORS restricted to your `FRONTEND_URL`
- For production, add rate limiting:
  ```bash
  pip install slowapi
  ```
